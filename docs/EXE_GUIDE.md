# GeekClock EXE 使用指南

> 一份给打包版（GeekClock.exe）用户的简明操作手册。不需要安装 Python，不需要配置环境，双击就能用。

---

## 一、快速开始

### 1.1 放置文件

把 `GeekClock.exe` 放到一个**固定文件夹**里（建议不放在桌面），例如：

```
D:\Apps\GeekClock\
```

> 程序会在 exe 同级目录创建 `logs/` 文件夹存放日志。不要直接在压缩包里双击运行——先解压出来。

### 1.2 首次启动

双击 `GeekClock.exe`，稍等几秒后会看到：

- **主窗口**：闹钟列表（首次为空）
- **任务栏右下角托盘图标**：一个时钟小图标

首次启动时 Windows 可能会弹出 SmartScreen 提示，见[常见问题](#61-windows-smartscreen-提示)。

### 1.3 退出程序

关闭主窗口（点 ×）**不会退出程序**，只是隐藏到托盘。闹钟继续在后台工作。

要真正退出：**右键托盘图标 → 退出 GeekClock**

---

## 二、数据存储

### 2.1 配置文件

闹钟数据、设置、用户昵称全部保存在：

```
C:\Users\<你的用户名>\AppData\Roaming\GeekClock\alarms.json
```

> 这是一个文本文件，可以用记事本打开查看（不建议手动编辑）。

### 2.2 日志文件

运行日志位于 exe 所在文件夹的 `logs\` 目录下：

```
D:\Apps\GeekClock\logs\GeekClock.log
```

每天一个文件，自动保留最近 7 天。不会撑爆硬盘。

### 2.3 为什么要分开放？

配置和 exe 分开存放，这样更新 exe 时闹钟数据不会丢失。

---

## 三、更新升级

1. 右键托盘图标 → **退出 GeekClock**
2. 用新的 `GeekClock.exe` 覆盖旧文件
3. 重新双击启动

闹钟数据在 `%APPDATA%\GeekClock\` 里，覆盖 exe 不会影响配置。

---

## 四、备份与迁移

### 4.1 备份闹钟数据

复制这个文件夹到安全位置：

```
C:\Users\<你的用户名>\AppData\Roaming\GeekClock\
```

恢复时把整个文件夹粘贴回原位置即可。

### 4.2 换电脑迁移

1. 旧电脑：退出程序 → 复制 `%APPDATA%\GeekClock\` 文件夹
2. 新电脑：先运行一次 GeekClock.exe 再退出 → 把备份的 `GeekClock\` 文件夹粘贴到 `%APPDATA%\` 下覆盖

---

## 五、开机自启

在程序内设置：**主窗口 → ⚙ 设置 → 勾选"开机自启动"**

原理：程序会在 Windows 注册表 `HKEY_CURRENT_USER\Software\Microsoft\Windows\CurrentVersion\Run` 中添加一条 `GeekClock` 记录，指向当前 exe 路径。

### 手动关闭开机自启

如果程序已卸载但注册表残留，可以手动操作：

1. 按 `Win + R`，输入 `regedit`，回车
2. 定位到 `HKEY_CURRENT_USER\Software\Microsoft\Windows\CurrentVersion\Run`
3. 删除 `GeekClock` 这一项

## 六、常见问题

### 6.1 Windows SmartScreen 提示

首次双击 exe 时，Windows 可能弹出"Windows 已保护你的电脑"。

**解决方法**：点击"更多信息"→"仍要运行"。这是独立开发者打包的常见现象，不是病毒。

> 如果介意这个提示，可以在文件属性中勾选"解除锁定"：右键 exe → 属性 → 常规 → 安全 → 解除锁定 → 确定。

### 6.2 杀毒软件误报

部分杀毒软件（如 360、腾讯电脑管家）可能将 PyInstaller 打包的 exe 误报为风险文件。

**解决方法**：
- 将 exe 所在文件夹添加到杀毒软件白名单
- 或在杀毒软件中选择"信任此文件"

### 6.3 双击没反应或闪退

排查顺序：

1. **检查托盘图标**：可能已经在运行，只是窗口隐藏了。看任务栏右下角有没有时钟图标。
2. **缺少 Visual C++ 运行时**：下载安装 [Visual C++ Redistributable](https://aka.ms/vs/17/release/vc_redist.x64.exe)（微软官方链接）
3. **查看日志**：打开 exe 所在文件夹的 `logs\` 目录，找到最新的 `.log` 文件，查看 `[ERROR]` 行

### 6.4 提示"已在运行"

表示已经有一个 GeekClock 实例在后台运行。检查任务栏右下角的托盘图标（可能需要点 `^` 展开）。如果没有看到托盘图标，打开任务管理器，找到 `GeekClock.exe` 进程并结束，然后重新启动。

### 6.5 闹钟到点没响

按顺序排查：

1. 主窗口里那个闹钟的开关是不是绿色（启用状态）
2. 当前时间在不在活动时段内
3. 当前是不是勿扰时段（设置 → 勿扰时段）
4. 选择的音频文件是否还存在

### 6.6 音响保活是什么？需要开吗？

定期播放静默音频，防止 Windows 音频设备因长时间不活动而休眠，导致闹钟不响。

如果你的电脑闲置一段时间后闹钟不响，建议开启。设为 0% 音量（完全静音），不会影响正常使用。

## 七、完全卸载

1. 右键托盘图标 → **退出 GeekClock**
2. 删除 `GeekClock.exe`
3. 删除配置文件夹（可选）：`C:\Users\<你的用户名>\AppData\Roaming\GeekClock\`
4. 检查开机自启注册表是否残留（见[第五章](#手动关闭开机自启)）

---

## 八、命令行参数

在 exe 所在目录打开命令行，可以用以下参数启动：

```cmd
GeekClock.exe --minimized    # 启动后最小化到托盘，不显示主窗口
```
